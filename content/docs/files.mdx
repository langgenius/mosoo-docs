---
title: "Files"
description: "Upload bytes and attach ready files to Mosoo API Threads."
---

Files added through the public API are Thread attachments. Files produced by the Agent are artifacts. Both appear in the Thread file list when visible to the API token caller.

## Upload flow

Public uploads are currently single PUT uploads up to 67108864 bytes.

1. Create a Thread-scoped upload session.
2. Upload bytes to the returned `fileId`.
3. Complete the upload.
4. Attach the ready file to the Thread.
5. Reference the file from a later user message with `attachmentIds`.

## Create an upload

```bash
curl -X POST "https://try.mosoo.ai/api/v1/threads/$MOSOO_THREAD_ID/files/uploads" \
  -H "Authorization: Bearer $MOSOO_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "file": {
      "name": "brief.txt",
      "contentType": "text/plain",
      "size": 41
    }
  }'
```

The response is a `FileUploadSummary`. Store `fileId`.

## Upload and complete bytes

```bash
curl -X PUT "https://try.mosoo.ai/api/v1/files/$MOSOO_FILE_ID/content" \
  -H "Authorization: Bearer $MOSOO_API_TOKEN" \
  -H "Content-Type: application/octet-stream" \
  --data-binary @brief.txt

curl -X POST "https://try.mosoo.ai/api/v1/files/$MOSOO_FILE_ID/complete" \
  -H "Authorization: Bearer $MOSOO_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{}'
```

## Attach to a Thread

```bash
curl -X POST "https://try.mosoo.ai/api/v1/threads/$MOSOO_THREAD_ID/files" \
  -H "Authorization: Bearer $MOSOO_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "fileId": "01J0000000000000000000000J"
  }'
```

The `POST /threads/{threadId}/files` endpoint attaches a ready uploaded file by `fileId`. It does not accept inline `contentBase64` payloads.

## Use an attachment

Send the file ID in `attachmentIds` on a later user message:

```json
{
  "events": [
    {
      "type": "user_message",
      "attachmentIds": ["01J0000000000000000000000J"],
      "text": "Summarize the attached file."
    }
  ]
}
```
