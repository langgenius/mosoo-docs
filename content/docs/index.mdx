---
title: "Mosoo Public Thread API"
description: "Call published Mosoo Agents from your application with HTTPS APIs."
---

The Mosoo Public Thread API lets your backend send work to a published Mosoo Agent and read the public results. Mosoo owns the Agent runtime, sandbox, model provider setup, tools, and execution lifecycle. Your application owns its product UI, business logic, user mapping, storage, and the saved `thread.id`.

<Callout type="info">
  The API only works with published Agents that have API access enabled. Agent setup, model provider connections, tool configuration, and channel bindings stay in Mosoo.
</Callout>

## First request

Set a Mosoo API token and a published Agent ID from Mosoo:

```bash
export MOSOO_API_TOKEN="mst_..."
export MOSOO_AGENT_ID="01J00000000000000000000001"
```

Create a Thread and queue the first Run:

```bash
curl -X POST "https://try.mosoo.ai/api/v1/agents/$MOSOO_AGENT_ID/threads" \
  -H "Authorization: Bearer $MOSOO_API_TOKEN" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: ticket-182-create-thread" \
  -d '{
    "client_external_ref": "ticket-182",
    "input": {
      "type": "user.message",
      "content": [
        {
          "type": "text",
          "text": "Triage this customer escalation and suggest next steps."
        }
      ]
    }
  }'
```

The response includes `thread.id` and, when `input` is present, the queued `run`. Use `thread.id` to send follow-up events, read event logs, stream updates, attach files, and manage lifecycle.
